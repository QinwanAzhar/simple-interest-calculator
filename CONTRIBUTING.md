# Contributing to Simple Interest Calculator

Thank you for your interest in contributing to this project! 🎉

All contributions, bug reports, bug fixes, documentation improvements, enhancements, and ideas are welcome.

## How to Contribute

### Reporting Bugs
If you find a bug, please open an issue with:
- A clear, descriptive title
- Steps to reproduce the problem
- Expected vs. actual behavior
- Your OS and Bash version

### Suggesting Enhancements
We love new ideas! Open an issue labeled `enhancement` and describe:
- The problem your idea solves
- How you envision it working
- Any alternatives you considered

### Submitting Pull Requests

1. **Fork** the repository and create your branch from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make your changes** – write clean, commented Bash code.

3. **Test your changes** thoroughly before submitting.

4. **Commit** with a clear message:
   ```bash
   git commit -m "feat: add support for monthly interest calculation"
   ```

5. **Push** to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```

6. Open a **Pull Request** against the `main` branch and fill in the PR template.

## Code Style Guidelines
- Use `#!/bin/bash` as the shebang line
- Add comments to explain non-obvious logic
- Use descriptive variable names (e.g., `principal` not `p`)
- Validate all user input

## Documentation Improvements
Found a typo or an unclear explanation? Documentation improvements are just as valuable as code changes. Feel free to submit a PR directly!

## Code of Conduct
Please note that this project is released with a [Code of Conduct](CODE_OF_CONDUCT.md). By participating in this project you agree to abide by its terms.

## Questions?
If you have any questions, feel free to open a discussion or contact the maintainers at [contribute@example.com].

We look forward to your contributions! 🚀
